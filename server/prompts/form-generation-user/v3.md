## SOURCE DOCUMENT LINKING INSTRUCTIONS

**CRITICAL: You MUST include `sourceIndex` in every bylaw reference.**

Each document above is labeled "SOURCE N" (e.g., SOURCE 1, SOURCE 2). When you cite a provision, include:
- `sourceIndex`: The number N from "SOURCE N" where the provision appears
- `pageNumber`: The page number if identifiable (e.g., "Page 42")
- `sectionId`: The section, article, or paragraph identifier (e.g., "Section 7.7A", "Article III")

This applies to:
- `relevantBylaws.primary` — set sourceIndex, pageNumber, sectionId
- Each entry in `relevantBylaws.additionalReferences` — set sourceIndex, pageNumber, sectionId
- Each field-level `relevantBylaws` — set sourceIndex, pageNumber, sectionId when citing a specific source

If a provision cannot be traced to a specific source (e.g., it's common knowledge), omit sourceIndex.

## MULTI-DOCUMENT INSTRUCTIONS

**You may have received MULTIPLE governing documents above** (covenants, design guidelines, addenda, board resolutions, etc.). Follow these rules:

1. **Cross-reference ALL sources for every topic.** Before generating any field or bylaw reference, check EVERY provided document for provisions related to that topic.
2. **When multiple documents address the same subject**, include references from ALL of them — in the field's `relevantBylaws`, in `complianceNotes`, and in any quotes. Cite each by its source name and section.
3. **Do NOT assume one document supersedes another.** If a design guideline says "no flags over 3ft" but an addendum says "American flags up to 6ft are permitted," include BOTH provisions and note the apparent override. Let the review committee decide.
4. **If documents are silent on a topic**, state that explicitly rather than guessing.

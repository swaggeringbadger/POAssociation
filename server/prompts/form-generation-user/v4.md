**CRITICAL: Include `sourceIndex` in every bylaw reference.**

## THE MOST IMPORTANT INSTRUCTION: BUILD STRUCTURED FIELDS FROM DOCUMENT DETAILS

Your primary goal is to produce a form with **maximum specificity**. Every time the documents give you a concrete list of options, those options MUST become a structured field — not a text box.

### Before creating ANY field, ask yourself:
1. Did the documents mention specific options for this? → **Use select/radio/checkbox with those options**
2. Did the documents give a number with limits? → **Use number field, cite the limits in description/bylaws**
3. Is this a yes/no decision? → **Use radio with Yes/No**
4. Did the documents truly provide NO specific guidance? → **Only then use text/textarea**

### Specific things to convert into structured fields:

**Materials** — If the documents list approved materials (for fences, siding, roofing, decking, paving, etc.), create a checkbox field (multi-select) with every listed material as an option. Example:
```json
{
  "id": "fence_material",
  "label": "Proposed Fence Material",
  "type": "checkbox",
  "required": true,
  "options": ["Cedar", "Composite", "Vinyl", "Wrought Iron", "Aluminum", "Chain Link (rear only)"],
  "relevantBylaws": {
    "reference": "Design Guidelines Section 7.7A",
    "requirement": "Only the following materials are approved for perimeter fencing",
    "sourceIndex": 1,
    "sectionId": "Section 7.7A"
  }
}
```

**Colors** — If the documents provide an approved color list, create a select dropdown with every color. If the list is very long (20+), still include all of them.

**Lot types** — If the documents define lot classifications, create a select or radio field with the exact types found. This is often one of the most important fields because other requirements vary by lot type.

**Styles/designs** — If the documents describe approved styles (e.g., "traditional, colonial, contemporary"), create a radio or select field.

**Height/size** — If the documents give a specific maximum or minimum, use a number field and state the limit in the description and relevantBylaws.

**Zones/areas** — If the documents define different zones or areas with different rules, create a select field for zone selection.

### Working with pre-extracted data

If the context includes JSON blocks with `enumerableLists`, `tables`, or `measurements` arrays (from a prior extraction stage), use them directly:
- `enumerableLists[].items` → field options
- `tables[].rows` → conditional requirements or dropdown options
- `measurements[].value` + `measurements[].condition` → field descriptions and bylaw references
